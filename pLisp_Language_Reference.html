Special operator abort

Syntax: (abort) => NIL

Arguments and Values: None

Description: Calling abort in debug mode aborts the evaluation of an expression.

Exceptional Situations: Calling abort when not in debug mode throws an exception of type EXCEPTION.

See Also: break

Macro alias

Syntax: (alias var1 var2) => NIL

Arguments and Values: var1, var2: symbols

Description: alias creates a new binding between var2 and the object bound to var1.

Exceptional Situations: Throws an exception of type SYMBOL-NOT-BOUND if var2 is not bound to a value.

Examples:

(define x 100)

(alias y x)

y=>100

See Also: define

Function alloc-ext-mem-char

Syntax: (alloc-ext-mem-char n) => (ptr, len, type)

Arguments and Values: n: integer, ptr: integer, len: integer, type: integer

Description: alloc-ext-mem-char allocates raw memory corresponding to n characters. Equivalent to malloc(n*sizeof(char)). Returns the pointer to the allocated memory via a list that also contains the number of characters allocated (i.e. n) and the type of the memory block (integer 3 signifying a character array).

Exceptional Situations: None

See Also: alloc-ext-mem-float, alloc-ext-mem-int, free-ext-mem, get-ext-mem-cell, inspect-ext-mem, set-ext-mem, set-ext-mem-cell

Returns the pointer to the allocated memory via a list that also contains the number of floats allocated (i.e. n) and the type of the memory block (integer 2 signifying a float array).

Function and

Syntax: (and form*) => result

Arguments and Values: form: an object; result: T or NIL

Description: and performs a logical conjunction of its operands. Returns T if all the operands evaluate to non-NIL and NIL otherwise. Returns T if no arguments are passed.

Exceptional Situations: None

Examples:

(and (eq 1 1) (eq 2 2)) => T

(and () 3) => NIL

See Also: or

Function append

Syntax: (append list1 list2) => result

Arguments and Values: list1, list2: forms that evaluate to lists; result: a list

Description: append creates a new list by appending the two lists passed as arguments.

Exceptional Situations: Throws an exception of type EXCEPTION if either of the arguments is not a list.

Examples:

(append '(a b) '(c d)) => (a b c d)

(append () '(1 2)) => (1 2)

See Also: concat, nconc

Special operator apply

Syntax: (apply function args) => result

Arguments and Values: function: a form that evaluates to a closure, a special operator or a continuation, args: a form that evaluates to a list; result: an object

Description: apply applies the closure, special operator or continuation passed as its first parameter to the arguments passed as its second parameter.

Exceptional Situations: Throws an exception of type INVALID-ARGUMENT if the first argument is not a closure, a special operator or a continuation or if the second argument is not a list.

Examples:

(apply + '(1 2)) => 3

See Also: funcall

Function array-eq

Syntax: (array-eq arr1 arr1) => result

Arguments and Values: arr1, arr2: forms that evaluate to array objects; result: T or NIL

Description: array-eq returns T if arr1 and arr2 are structurally equivalent, i.e. they have the same contents, and NIL otherwise.

Exceptional Situations: Throws an exception of type INVALID-ARGUMENT if either of the arguments is not an array.

Examples:

a1 => [1, 2]

a2 => [1, 2]

a3 => ['a 'b]

(array-eq a1 a2) => T

(array-eq a1 a3) => NIL

See Also: eq

Special operator array-get

Syntax: (array-get arr index) => result

Arguments and Values: arr: a form that evaluates to an array object; index: a form that evaluates to an integer; result: an object

Description: array-get returns the object at the position index of array arr. Note that array indices start from zero.

Exceptional Situations: Throws an exception of type INVALID-ARGUMENT if the first argument is not an array, and an exception of type INDEX-OUT-OF-BOUNDS if the index is less than zero or greater than (len-1), where len is the length of the array.

Examples:

a1 => [1, 2]

(array-get a1 0) => 1

See Also: array-set

Special operator array-length

Description: array-length returns the length of array arr.

Exceptional Situations: Throws an exception of type INVALID-ARGUMENT if the first argument is not an array.

Examples:

a1 => [40, 50]

(array-length a1) => 2

See Also:

Special operator array-set

Syntax: (array-set arr index form) => form

Arguments and Values: arr: a form that evaluates to an array object; index: an integer; form: an object

Description: array-set binds the position index of array arr to form.

Exceptional Situations: Throws an exception of type INVALID-ARGUMENT if the first argument is not an array, and an exception of type INDEX-OUT-OF-BOUNDS if the index is less than zero or greater than (len-1), where len is the length of the array.

Examples:

a1 => [40, 50]

(array-set a1 0 30) => 30

a1 => [30, 50]

See Also: array-get

Syntax: (caar lst) => result

Arguments and Values: lst: a form that evaluates to a list; result: an object

Special operator call-foreign-function

Syntax: (call-foreign-function name return-type params) => result

Arguments and Values: name: a form that evaluates to a string; return-type: a symbol; params: a list of CONS objects; result: an object

Description: call-foreign-function invokes a native function called name defined in a shared library that was loaded earlier. return-type denotes the return type of the native function ('void, 'integer, 'float, 'character, or 'character-pointer). params contains information about the parameters to the native function (parameter type and object).

Exceptional Situations: call-foreign-function throws an exception of type EXCEPTION if it is not supplied with three parameters. It throws an exception of type ARG-MISMATCH if the parameters are not of the types as described above. ARG-MISMATCH is also thrown if there are any inconsistencies in the params argument (e.g. the parameter type is integer and the object is not an integer).

Examples:

(call-foreign-function some_func 'void '((ivar integer, fvar float)))

See Also: load-foreign-library

Macro cond

Syntax: (cond ((cond1 action1) (cond2 action2) ..)) => result

Arguments and Values: cond1, cond2, : forms; action1, action2, : forms; result: an object

Description: cond takes a list comprising condition/action pairs, and evaluates each condition sequentially, stopping at the first condition that evaluates to a non-NIL value. It then evaluates the action corresponding to this condition and returns the result of evaluating that action. If none of the conditions evaluates to a non-NIL value, cond returns NIL. If the last condition is T, the action corresponding to this serves as the default action.

Exceptional Situations: None

Examples:

(cond ((eq 1 1) 'true) (t 'false))) => TRUE

See Also: IF

Function cons-pair

Syntax: (cons-pair list1 list2) => result

Arguments and Values: list1, list2: forms that evaluate to lists; result: a list

Description: cons-pair takes two lists and creates a new list, the elements of which are CONS objects obtained by applying cons on the matched elements of the two lists. If the sizes of the two lists are not the same, the shorter of the two lists is padded with NIL values.

(cons '(a b c) '(1 2)) => ((A . 1) (B . 2) (C))

See Also:

(g 30) => 60

Macro define-class

Syntax: (define-class class-name superclass-name class-var-list instance-var-list class-init-form instance-init-form class-methods instance-methods) => result

Arguments and Values: class-name: symbol; superclass-name: symbol; class-var-list: a list; instance-var-list: a list; class-init-form: a list; instance-init-form: a list; class-methods: a list; instance-methods: a list result: an object

Description: define-class creates a pLisp Object System (POS) class named class-name that is derived from a superclass superclass-name. class-var-list specifies the class variables, getters and setters for the class, while instance-var-list specifies the same entities for an instance of the class. class-init-form is the form evaluated as the class initializing code, while instance-init-form serves the purpose of constructor. Class-methods and instance-methods specify the methods at the class level and the instance level respectively.

Exceptional Situations: None

Examples:

(define-class 'square 'shape nil '((a get-a set-a)) nil (lambda (var) (set a var)) nil ((area (lambda () (* a a)))) => <closure denoting the class>

See Also:

Macro dolist

Description: dolist evaluates the code in body for each element in the list specified in the CDR of spec, with the value of the element bound to the symbol specified in the CAR of spec. It returns the result of the last evaluation of body.

Exceptional Situations: None

Examples:

2

3

NIL

See Also:

Macro dotimes

Syntax: (dotimes spec &rest body) => result

Arguments and Values: spec: a list; body: the rest of the arguments converted into a list; result: an object

Description: dotimes initializes the variable specified in the CAR of spec to zero, and evaluates body with the variable bound to this value. It then successively increments the variable and evaluates body till the value of the variable exceeds the value specified in the CDR of spec. It returns the value of the last evaluation of body.

Exceptional Situations: None

Examples:

(dotimes (x 3) (print x)) =>

0

1

2

NIL
